## 🔴 原始问题

用户反馈：**AI 不能真正连续处理问题，非常容易断线**

### 问题分析
1. 连续执行逻辑有 Bug（变量名冲突）
2. API 调用无保护机制（无超时、无重试）
3. JSON 解析容错性差
4. 上下文管理缺失（容易达到 token 限制）
5. 缺少用户交互确认机制

## 🎯 新功能：继续/停止确认

### 触发条件
- 达到最大迭代次数（默认 10 次）
- AI 回复不包含完成标志

### 用户选项
```
═══════════════════════════════════════════════════════════
  ⚠️  任务尚未完成，是否继续执行？
═══════════════════════════════════════════════════════════

  已执行迭代：10 次
  当前任务：帮我分析...

  选择：
    [1] 继续执行（让 AI 继续完成任务）
    [2] 停止执行（查看当前结果）
    [3] 增加迭代次数（设置新的最大值）
    [4] 查看详细历史

  请选择 [1-4]:
```

### 完成标志词
AI 回复包含以下词汇时，认为任务已完成：
- "完成"
- "总结"
- "分析完毕"
- "已完成"

## 💡 最佳实践

### 1. 合理设置迭代次数
- 小任务：3-5 次
- 中等任务：5-10 次
- 大任务：10-20 次
- 超大任务：20-50 次

### 2. 利用并行调用
AI 可以一次返回多个工具调用，例如：
```json
[
  {"tool": "fm", "command": "read", "args": ["file1.txt"]},
  {"tool": "fm", "command": "read", "args": ["file2.txt"]},
  {"tool": "fm", "command": "read", "args": ["file3.txt"]}
]
```

### 3. 监控迭代进度
系统会显示：
```
[迭代 3/10] AI 正在思考...
```

### 4. 使用完成标志
任务完成后，AI 会明确说明。
