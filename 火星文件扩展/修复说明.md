# 火星编程平台修复说明

## 🔴 问题分析

之前火星编程平台存在以下问题，导致 AI 不能连续处理问题且容易断线：

### 1. 连续执行逻辑 Bug
- **变量名冲突**：连续调用时使用了错误的变量 `toolCalls` 而不是 `nextToolCalls`
- **错误处理缺失**：连续执行循环中没有 `try-catch`，一次失败就中断整个流程
- **迭代次数过少**：最大连续调用次数只有 5 次，复杂任务不够用

### 2. API 调用无保护机制
- **无超时控制**：API 调用可能无限期等待
- **无重试机制**：网络错误直接失败
- **无退避策略**：失败后立即重试，可能加重服务器负载

### 3. JSON 解析容错性差
- 只能解析标准 ` ```json``` 代码块
- AI 返回格式稍有变化就无法识别工具调用

### 4. 上下文管理缺失
- 历史记录无限制增长，容易达到 token 限制
- 没有压缩机制，复杂任务容易超时/断线

## ✅ 修复内容

### 1. 增强 `callAI` 函数（带重试和超时）

```javascript
async function callAI(messages, maxRetries = 2, timeout = 60000)
```

**新增功能：**
- ✅ 超时控制（默认 60 秒）
- ✅ 自动重试（最多 2 次）
- ✅ 指数退避（1 秒 → 2 秒 → 4 秒）
- ✅ 详细日志输出

### 2. 增强 `callMcpServer` 函数

```javascript
async function callMcpServer(serverName, method, params, maxRetries = 2)
```

**新增功能：**
- ✅ 超时控制（60 秒）
- ✅ 自动重试机制
- ✅ 错误处理和恢复

### 3. 修复 `parseToolCall` 函数（增强容错性）

**新增解析策略：**
1. 标准 JSON 代码块 ` ```json``` `
2. 无标记代码块 ```` ``
3. 内联 JSON 对象（ brace 匹配）
4. JSON 数组（ bracket 匹配）

### 4. 修复连续执行逻辑

**主要修复：**
```javascript
// 修复前：使用错误的变量
toolCalls.forEach((tc, i) => { ... });

// 修复后：使用正确的变量
nextToolCalls.forEach((tc, i) => { ... });
```

**改进：**
- ✅ 最大迭代次数从 5 次增加到 10 次
- ✅ 添加完整的错误处理
- ✅ 工具执行失败后继续让 AI 处理
- ✅ 达到最大迭代次数时给用户提示

### 5. 新增上下文管理机制

```javascript
const CONTEXT_CONFIG = {
  maxMessages: 50,        // 最大消息数量
  compressThreshold: 30,  // 超过此数量开始压缩
  keepRecent: 10,         // 保留最近的消息不压缩
};
```

**功能：**
- ✅ 自动压缩早期历史消息
- ✅ 保留最近的对话内容
- ✅ 防止达到 token 限制
- ✅ `prepareMessages()` 函数优化消息列表

### 6. 优化系统提示词

**新增提示：**
- ✅ 错误处理指导：遇到错误不要放弃
- ✅ 连续执行指导：主动拆解任务
- ✅ 多方法尝试：提供替代方案
