# 火星编程 CODE - 使用指南

## 快速开始

### 启动平台
```bash
cd C:\Users\Administrator\Desktop\火星文件扩展
node mars-platform.js
```

### 基本命令

| 命令 | 说明 |
|------|------|
| `/help` | 显示帮助信息 |
| `/log` | 查看最近 50 行聊天日志 |
| `/log <行数>` | 查看指定行数的日志 |
| `/clearlog` | 清空聊天日志 |
| `/setconmode` | 设置控制台缓冲区 |
| `/history` | 查看 AI 历史记忆 |
| `/quit` | 退出平台 |

## 核心功能

### 1. AI 对话编程
直接输入你的编程需求，AI 会自动分析并执行工具调用。

**示例**：
```
❯ 帮我优化 caihang.html 的视觉效果
```

### 2. 火星文件管理
使用 fm 命令前缀执行文件操作：

**示例**：
```
❯ fm tree src --depth 2
❯ fm read index.html -s 1 -e 50
```

### 3. 连续执行
AI 可以连续调用工具完成复杂任务，无需人工干预。

### 4. 继续/停止确认
当任务执行到第 9 次迭代时，系统会弹出确认菜单：
- 按 **B** 继续执行
- 按 **N** 停止

### 5. 聊天日志
所有对话自动保存到 `chat-log.txt`，支持：
- 日志轮转（最大 1MB）
- 保留 5 个备份
- 使用 `/log` 命令查看

### 6. 控制台滑动
控制台缓冲区自动设置为 9999 行，支持：
- 鼠标滚轮滑动
- 窗口滚动条
- PageUp / PageDown 键

## AI 工具调用格式

AI 使用 JSON 格式调用工具：

```json
{"tool": "fm", "command": "patch", "args": ["文件路径", "原文本", "新文本"]}
```

### 支持的命令
- `read` - 读取文件
- `tree` - 查看目录树
- `search` - 搜索文本
- `patch` - 修改文件（自动转换为 apply 命令）
- `apply` - 应用编辑操作

## 故障排除

### 问题：AI 调用失败
**解决**：检查 API Key 配置
```bash
# 设置环境变量
set AI_API_KEY=sk-xxxxx
```

### 问题：patch 命令失败
**原因**：原文本不匹配
**解决**：使用更短的原文本，或改用 test-ai-fuzzy.js 脚本

### 问题：控制台无法滑动
**解决**：
```bash
# 在平台内执行
/setconmode
```

### 问题：日志文件太大
**解决**：
```bash
# 在平台内执行
/clearlog
```

## 配置文件

### 环境变量
| 变量 | 说明 | 默认值 |
|------|------|--------|
| `FM_WORKSPACE` | 工作区路径 | 当前目录 |
| `AI_API_KEY` | AI API 密钥 | 空 |
| `AI_PROVIDER` | AI 提供商 | dashscope |
| `AI_MODEL` | AI 模型 | qwen3.5-plus |
| `AI_BASE_URL` | API 地址 | 阿里云 DashScope |

## 最佳实践

1. **复杂任务**: 让 AI 自动连续执行，无需干预
2. **文件修改**: 使用 patch 命令，AI 会自动处理版本
3. **查看历史**: 使用 `/log 100` 快速查看最近内容
4. **搜索内容**: 打开 `chat-log.txt` 使用文本编辑器搜索
5. **管理日志**: 定期使用 `/clearlog` 清理旧日志
