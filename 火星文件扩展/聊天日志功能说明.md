# 聊天日志功能 - 使用说明

## 🔴 问题解决

**问题**：命令行窗口聊天内容多时，无法滑动查看之前的内容

**解决方案**：
1. ✅ 自动增加控制台缓冲区到 9999 行
2. ✅ 聊天记录自动保存到文件
3. ✅ 提供 `/log` 命令查看历史
4. ✅ 提供 `/setconmode` 手动设置缓冲区

---

## 📋 使用方法

### 方法 1：使用 /log 命令（推荐）

```
❯ /log              # 查看最近 50 行日志
❯ /log 100          # 查看最近 100 行日志
❯ /log 500          # 查看最近 500 行日志
```

### 方法 2：直接打开日志文件

日志文件位置：
```
C:\Users\Administrator\Desktop\火星文件扩展\chat-log.txt
```

可以用任何文本编辑器打开查看：
- 记事本
- Notepad++
- VS Code
- 等等

### 方法 3：手动滑动控制台窗口

启动平台后，控制台会自动设置缓冲区大小：
- 宽度：120 列
- 高度：9999 行

如果无法滑动，手动设置：
```
❯ /setconmode
```

### 方法 4：清空日志

```
❯ /clearlog         # 清空聊天日志
```

## 📊 日志功能

### 自动保存
- 所有聊天内容自动保存到 `chat-log.txt`
- 包含时间戳
- 去除 ANSI 颜色码（干净文本）

### 日志轮转
- 最大文件大小：1MB
- 超出后自动轮转
- 保留最近 5 个日志文件

文件命名：
```
chat-log.txt           (当前日志)
chat-log.txt.1.txt     (最近的备份)
chat-log.txt.2.txt     (较旧的备份)
...
chat-log.txt.5.txt     (最旧的备份)
```

### 日志格式
```
[2026-03-13 11-30-45] ════════════════════════════════════
[2026-03-13 11-30-45]   火星编程 CODE - 帮助文档
[2026-03-13 11-30-45] ════════════════════════════════════
[2026-03-13 11-30-50] 🚀 火星编程 CODE ❯ 帮我查看 src 目录
[2026-03-13 11-30-52]   AI 正在思考...
[2026-03-13 11-30-55]   正在执行：fm tree src --depth 2
```

## ⚙️ 配置说明

### 修改日志配置

在 `mars-platform.js` 中修改：

```javascript
const LOG_CONFIG = {
  enabled: true,              // 是否启用日志
  maxFileSize: 1024 * 1024,   // 最大文件大小 (1MB)
  backupCount: 5,             // 保留的备份数量
};
```

### 修改缓冲区大小

在 `setupConsoleBuffer` 函数中修改：

```javascript
exec('mode con cols=120 lines=9999', (error) => {
  // cols: 宽度（列数）
  // lines: 高度（行数）
});
```

## 🔍 故障排除

### 问题 1：无法滑动窗口

**解决**：
```
❯ /setconmode
```

或者手动设置：
1. 右键点击标题栏
2. 选择"属性"
3. 切换到"布局"选项卡
4. 增加"屏幕缓冲区大小"的高度

### 问题 2：日志文件不存在

**原因**：还没有聊天内容

**解决**：开始对话后自动创建

### 问题 3：日志文件太大

**解决 1**：清空日志
```
❯ /clearlog
```

**解决 2**：手动删除旧日志
```
删除 chat-log.txt.5.txt（最旧的备份）
```
